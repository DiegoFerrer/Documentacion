
<!--               Programacion Funcional                  -->


<script>
	/* 

	//					Caracteristicas funcionales
	• Arquitecturas centradas en la transformacion
	• Inmutabilidad (no cambia los datos si no que genera uno nuevo )
	* Evita todo los iteradores comunes, ya que estos mutan los datos
	• Compositividad
	• Variantes Funcionales
	• Transparencia referencial
	• Arquitecturas dirigidas por flujos de datos

	 //					 Programacion declarativa   

	•	Indica el que: describe un problema en términos de proposiciones y 			afirmaciones, sin especificar los pasos para resolverlo
	•	Evitar cambiar el estado de un programa al prevenir la mutación de valores
	•	Usa funciones puramente matemáticas que carecen de efectos secundarios
	•	No existen estructuras cíclicas, la forma de iterar es la recursión de		funciones de alto nivel como map, reduce y filter
	•	No hay asignación destructivas de variables ,por lo general una variable 	 no puede ser asignada más de una vez
	•	Las variables pasadas transparentemente de una función a otra
	•	Código más expresivo y relativamente más corto

	*/

	// Codigo Declarativo

	const numbers = [1,2,3,4,5]

	numbers.map(num => num * num)

	// Metodos funcionales: map, filter y reduce
	// reciben como parametro una =>, tiene ambito de bloque y evita iteraciones


</script>
